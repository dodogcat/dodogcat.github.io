### 요약

Unity 기반 모바일 환경에서 객체 인식 기능을 개선하기 위해 기존 **YOLOv3-tiny** 구조를 분석하고, **YOLOv8 계열로의 전환 가능성**을 검토한 프로젝트입니다.  
계측 결과, 체감 성능 저하의 주요 원인은 모델 자체보다 **후처리 단계와 Unity 코드 흐름**에 더 가까웠습니다.

새 모델 이식은 제안하고 검토했지만, 런타임 호환성 문제로 인해 실제 구조를 완전히 변경하지는 않았습니다.

### 문제

- 기존 구조에서는 원하는 수준의 인식 품질을 확보하기 어려웠습니다.
- YOLOv8n 적용을 검토하는 과정에서 화면 끊김이 발생했고, 원인이 모델 크기인지 파이프라인 구조인지 분리해서 확인할 필요가 있었습니다.

### 결과

![단계별 시간 측정 결과](/images/notion5.png)
*입력 크기와 모델 구성에 따른 단계별 시간 측정 결과*

- YOLOv8n, YOLOv8s, YOLOv8m 비교 결과, 총 추론 시간은 **67ms, 88ms, 147ms**였지만 평균 프레임은 **40, 38, 38 FPS**로 큰 차이가 없었습니다.
- **YOLOv8n(416)**에서는 전처리 4ms, 추론 4ms, 후처리 59ms였고, **YOLOv8n(640)**에서는 후처리가 108ms까지 증가했습니다.
- 주요 병목이 모델 본체보다 **후처리와 Unity 코드 흐름**에 있다는 점을 확인했습니다.
- 모바일 추론 파이프라인을 단계별로 분석할 수 있는 계측 구조를 만들었습니다.
